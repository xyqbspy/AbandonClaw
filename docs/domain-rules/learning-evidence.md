# Learning Evidence

## 1. 目标

这份文档用于定义项目里的“学习证据”边界，也就是哪些行为可以被视为真实学习推进，哪些只能算局部动作或临时 UI 状态。

## 2. 基本原则

- 学习证据必须能被稳定消费，而不是只停留在前端瞬时状态
- 单次进入页面、单次点击或单次浏览，不应直接等价成学习完成
- 场景步骤推进、句子完成、场景练习完成、场景 done 必须是不同层级的证据

## 3. 当前稳定证据层级

### 3.1 场景步骤证据

例如：

- `full_play`
- `open_expression`
- `practice_sentence`

这些行为表示用户已经推进到某个训练阶段，但不直接等价于句子或场景完成。

### 3.2 句子里程碑证据

句子完成必须以句子最高里程碑达到 `complete` 为准。

这意味着：

- `keyword`
- `structure`

都属于句子里程碑升级，但还不是句子完成。

### 3.3 场景练习完成证据

只有当前场景的必做题型全部完成，才能把 `scene_practice` 视为完成。

### 3.4 场景收束证据

`done` 高于 `scene_practice`，只有在练习完成且下游条件满足后，才允许进入 `done`。

## 4. Review 里的证据边界

`review` 里要区分：

- 前端阶段式训练状态
- 正式后端信号

只有已经正式落库并被聚合消费的信号，才属于稳定学习证据。

## 5. 维护约束

- 不得把“进入练习”直接近似为“句子已完成”
- 不得把本地临时阶段误写成正式学习完成
- today / progress / review 只能消费稳定聚合字段，不应各自发明新的证据解释

## 6. 改动时一起检查

- `learning-loop-overview` 主规范
- `sentence-progression` 规范
- `sentence-completion-tracking` 规范
- scene / review / today 的聚合消费逻辑
