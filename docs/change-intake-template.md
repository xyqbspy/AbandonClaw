# 变更接入模板

在开始任何非微小改动前，先用这份模板快速过一遍。

## 1. 变更摘要

- 变更名称：
- 目标模块：
- 变更类型：
- 功能行为
- 数据流 / 状态流
- 缓存
- 测试
- 跨页面 UI 一致性
- 维护规范

## 2. 完整链路检查

- 上游入口是什么？
- 当前节点承担什么职责？
- 下游会写回哪里？
- 失败、回退、空态会落到哪里？
- 这次改动会不会影响 `today -> scene -> chunks -> review` 中的某一段？

## 3. 功能连续性检查

- 如果只改局部，这条链路会不会在别处断掉？
- 是否存在状态不同步风险？
- 是否存在按钮、文案、禁用态、跳转路径不一致风险？
- 是否需要拆成多步改动，避免一次改太多？

## 4. 测试影响检查

- 现有哪些测试会直接受影响？
- 现有测试是否需要更新？
- 是否需要新增回归测试？
- 如果这次不补测试，剩余风险是什么？

## 5. OpenSpec 判断

- 这次是否应该先建 change？
- 如果建 change：
- change 名称是什么？
- proposal 要覆盖哪些影响范围？
- delta spec 要改哪条 capability？

## 6. 收尾要求

- 是否更新 `CHANGELOG.md`
- 是否记录验证结果
- 是否满足 archive 前检查条件
