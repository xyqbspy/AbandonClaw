# AbandonClaw 产品说明总览

## 1. 产品定位

AbandonClaw 是一个面向中文学习者的英语学习系统。

它不是传统的“单词卡片”产品，也不是只做内容浏览的课程页，而是一个围绕真实语境、表达沉淀和持续回流构建的学习闭环产品。

一句话概括：

> 这是一个以场景学习为主线、以表达资产沉淀为核心、以每日任务编排为入口的英语学习系统。

## 2. 产品想解决的问题

当前英语学习产品常见有三类问题：

- 学习入口碎片化，用户不知道今天该先做什么
- 学习内容只看不沉淀，学过的表达无法形成个人资产
- 学习状态停留在前端体验层，没有形成稳定的长期进度闭环

AbandonClaw 的目标不是只让用户“看完一个内容”，而是解决下面这条链的连续性问题：

1. 今天从哪里开始学
2. 在真实语境里学什么
3. 学到的表达怎么沉淀
4. 沉淀后的内容如何复习和回流
5. 第二天系统如何继续接住用户

## 3. 目标用户

当前产品更适合以下用户：

- 希望通过真实场景而不是纯词表学习英语的中文学习者
- 已经有一定基础，但缺少持续输入、输出和复习闭环的人
- 希望逐步建立个人表达库，而不是只完成题目的人
- 需要碎片化进入，但又希望系统能记住自己学到哪的人

## 4. 核心产品理念

### 4.1 场景驱动，而不是词库驱动

用户的学习起点不是孤立单词，而是一个具体场景。

用户先进入语境，再在语境中：

- 阅读内容
- 听音频
- 识别重点表达
- 做练习
- 看变体

这使学习更接近真实使用环境，而不是脱离语境的死记硬背。

### 4.2 表达资产化，而不是只读不存

场景中的 chunk、phrase 不只是“看过”，还会进入用户自己的表达资产库。

这些资产后续可以继续：

- 补全翻译和例句
- 建立相似表达 / 对照表达
- 组织成 expression cluster
- 加入复习链路

也就是说，学习结果不是一次性消耗，而是可持续积累。

### 4.3 每日编排，而不是功能平铺

产品不是把所有能力直接摆给用户，而是通过 `today` 页把底层学习数据重新编排成“今天该做什么”。

这能降低用户决策成本，让系统承担任务组织角色，而不是把选择压力留给用户。

### 4.4 服务端学习闭环，而不是本地瞬时状态

项目中的学习状态不是简单的前端点击记录，而是由服务端维护完整闭环：

- 开始
- 推进
- 暂停
- 完成

这保证了：

- 学习可以恢复
- 进度可以聚合
- today / review / progress 能消费稳定结果

## 5. 产品整体结构

当前产品可以理解为四层结构：

### 5.1 Today：每日任务入口层

负责回答：

- 今天最应该先做什么
- 当前学习卡在什么位置
- 今天还有哪些下游任务可做

它本质上是任务编排层，不是内容生产层。

### 5.2 Scene：主学习流程层

负责承载一次完整的场景学习。

用户在这里完成：

- 阅读和听熟场景
- 查看重点表达
- 进入练习
- 解锁变体
- 推进学习进度

这是产品的主流程层。

### 5.3 Chunks：表达资产层

负责把用户在学习过程中沉淀出的表达转成长期可用资产。

核心能力包括：

- 搜索和筛选表达
- 查看详情
- AI 补全
- 关系维护
- 聚类管理
- 进入复习

这是产品的资产中台。

### 5.4 Review / Progress：复习与概览层

`review` 负责把沉淀后的内容重新拉回回忆链路，并写回正式学习信号。  
`progress` 负责展示长期学习表现和概览统计。

这两者一起承担“复习闭环”和“长期反馈”。

## 6. 当前核心学习闭环

当前产品主闭环可以概括为：

1. 用户进入 `today`
2. 系统推荐继续哪个 `scene` 或今天先做什么
3. 用户进入 `scene` 做主学习
4. 在场景中保存表达，沉淀到 `chunks`
5. 表达进入 `review`，产生正式训练信号
6. 服务端聚合学习状态、复习结果和长期概览
7. `today` 第二天继续重新组织入口

一句话表示：

> Today 分发任务 -> Scene 完成主学习 -> Chunks 沉淀资产 -> Review 写回信号 -> Learning Service 聚合 -> Today 再次编排入口

## 7. 当前主要页面与用户价值

### 7.1 Today 页

用户价值：

- 快速开始今天的学习
- 找到上次中断的位置
- 看到复习和表达沉淀的任务状态

### 7.2 Scenes 列表页

用户价值：

- 选择要进入的学习场景
- 导入或生成新的学习场景
- 快速进入下一个学习内容

### 7.3 Scene 详情页

用户价值：

- 在真实语境中完成主学习
- 从阅读走向练习与变体
- 在学习中提取值得反复学习的表达

### 7.4 Chunks 页

用户价值：

- 管理自己的表达资产
- 继续补全、整理、分类和关联表达
- 把零散学习痕迹变成长期资产

### 7.5 Review 页

用户价值：

- 把已经学过的表达重新拉回回忆链路
- 形成正式训练信号
- 调整后续复习节奏

### 7.6 Progress 页

用户价值：

- 看到近期学习表现
- 理解自己是否保持连续学习
- 感知长期积累是否在增长

## 8. 当前关键能力

当前产品已经具备的关键能力包括：

- 场景列表加载、导入、生成与进入前预热
- 场景学习中的阅读、听音频、重点表达查看、练习和变体
- 场景进度同步与 session 恢复
- 表达保存、手动录入、AI 补全、相似/对照表达维护
- expression cluster 组织能力
- review 正式信号写回与调度细化
- today 聚合入口与 continue learning 编排
- progress 长期概览聚合
- 前端缓存命中 + 后台刷新 + 服务端稳定聚合

## 9. 当前产品边界

当前产品已经形成闭环，但仍处于“闭环骨架已跑通、体验可继续增强”的阶段。

当前边界包括：

- `today` 更偏聚合入口，推荐策略还可以更强
- `chunks` 已是重型工作台，但仍有继续拆薄空间
- `review` 已有正式信号，但更细粒度训练反馈仍可扩展
- `scene` 的进度估算和训练颗粒度仍可进一步优化

所以它当前最准确的定位是：

> 一个已经跑通主学习闭环、具备表达资产沉淀能力、并开始形成复习与聚合反馈体系的英语学习产品。

## 10. 当前产品优势

- 主线清楚：`today -> scene -> chunks -> review/progress`
- 学习不只停留在浏览，而是有沉淀和回流
- 表达资产不是平铺存储，而是有关系和 cluster 组织
- 学习状态由服务端维护，支持恢复和长期聚合
- 页面职责边界和系统文档已经比较清楚，利于继续演进

## 11. 后续可演进方向

- 加强 `today` 的推荐策略和任务解释能力
- 增强 `review` 的训练反馈细粒度和调度智能度
- 继续强化 `chunks` 的资产运营能力
- 优化 `scene` 中训练颗粒度与进度评估
- 在保持闭环一致的前提下，增强个性化学习路径

## 12. 一页版总结

AbandonClaw 的核心不是“做了几个学习页面”，而是已经把英语学习里的几个关键环节串成了一条连续链路：

- 用 `today` 降低每天开始学习的决策成本
- 用 `scene` 提供真实语境中的主学习流程
- 用 `chunks` 把表达沉淀成长期资产
- 用 `review` 把资产重新拉回复习闭环
- 用 `progress` 和聚合服务给出长期反馈

它当前最有价值的地方，不是单一页面或单一 AI 能力，而是：

> 它已经具备“场景学习 -> 表达沉淀 -> 正式回写 -> 聚合反馈 -> 再次进入”的完整产品骨架。
