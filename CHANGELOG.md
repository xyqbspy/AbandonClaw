# Changelog

只记录用户可感知变化。开发过程、验证结果和维护收尾记录放在 `docs/dev/dev-log.md`。

## 2026-05-14

### Chunks 必备表达
- `/chunks` 新增“我的表达 / 必备表达”双入口，新用户即使还没有保存个人表达，也可以先浏览系统内置高频表达。
- 必备表达来自默认 starter / builtin scenes，支持按等级和分类筛选，并展示中文解释、来源场景、等级、分类和保存按钮。
- 用户点击“保存到我的表达”后，表达才会进入个人表达资产和复习队列；浏览必备表达不会自动污染个人资产。

### Scenes 入门内容与移动端学习入口
- 新用户现在可以在 `/scenes` 直接看到一组内置日常入门场景，不需要先导入或生成内容才能开始学习。
- `/scenes` 首屏改为移动端优先的学习入口：推荐路径、筛选与操作区、继续学习卡片和底部主 CTA 会优先服务开始/继续学习。
- 场景列表支持按分类、等级、来源筛选，并保留生成、导入、删除和循环复习能力，但这些操作不再压过学习主路径。

### Today 新用户推荐与移动端视觉收口
- 新用户第一次进入 `/today` 现在会优先看到基于 builtin starter scenes 的“今天从这里开始”推荐，不再出现空状态或只有后台式入口。
- `today` 会按“继续学习 -> 新手 starter -> 下一组日常场景”的稳定路径推荐下一步，卡片会同时显示推荐理由、难度、预计时长、进度和清晰 CTA。
- `today` 移动端页面同步按新视觉组件收口欢迎卡、继续学习主卡、表达库、复习状态和“推荐下一组”区块，移动端入口层级和文字可读性更清晰。
## 2026-05-13

### 注册邮箱验证链路收口
- 注册成功链路收口为项目内 6 位邮箱验证码：用户完成验证码校验并创建账号后，不再需要额外点击 Supabase Confirm email 才能进入主应用。
- 登录失败时会展示更清晰的中文提示，仍不会拆分邮箱是否存在或密码是否错误。

## 2026-05-12

### Settings 与表达库体验收口
- `settings` 页面按新的设置中心视觉重做，昵称和邮箱改为展示当前登录用户真实信息；发音速度偏好保存后会影响 TTS 播放速度，其余暂未接入的学习偏好保持只读展示。
- `chunks` 表达库继续收口卡片视觉；无同类表达时不再展示展开入口和空翻译提示，同类表达入口以置灰状态展示，减少误点击。

### 主应用界面与后台体验收口
- 主应用顶部栏和移动端菜单重做，新增全局面包屑展示，让用户在 `admin`、`progress`、`demo` 等页面能更清楚当前位置。
- `today` 页面简化为更直接的学习入口，降低卡片和说明堆叠带来的启动成本。
- `progress` 页面重做为学习成就中心，保留学习热力图和成长足迹，并把核心指标收口为连续学习、表达资产、累计时长和复习正确率。
- `demo` 页面重做为更贴近产品真实学习闭环的展示页。
- 管理后台首页和 `/admin/invites` 继续统一为后台专属视觉风格，保留用户、邀请码、场景、表达库、导入场景、变体、AI 缓存、TTS 缓存和可观测性入口。
- 应用内路由切换和后台长等待操作新增加载反馈；后台 server action 提交按钮会在处理中禁用并显示加载态，降低重复点击和慢请求时的误操作。

## 2026-05-11

### 注册邮箱验证码
- 注册页新增邮箱验证码流程：用户需要先发送 6 位验证码，再填写验证码完成注册；邀请注册模式下仍需同时填写邀请码。
- 注册接口会在创建账号前校验验证码，错误、过期或已消费验证码不会创建账号，也不会扣减邀请码次数。

### 注册入口与后台处置继续收口
- `/api/auth/signup` 现在会在邀请码校验和账号创建前按同一 IP 频控；命中后直接返回受控 429，并带 `requestId`。
- 管理后台新增 `/admin/users`，管理员可以按邮箱、用户 ID、用户名和账号状态查找用户，并直接切换 `active`、`disabled`、`generation_limited`、`readonly`。
- 管理后台新增 `/admin/invites`，管理员可以生成、停用和调整邀请码，并查看邀请码被哪个邮箱/账号使用及最小活动摘要。
- 管理后台新增注册模式切换，可以在 `/admin/invites` 里切换关闭注册、邀请注册和公开注册；后台配置优先生效，环境变量仍作为兜底。
- 管理后台新增高成本紧急开关，可临时关闭练习生成、场景生成、解释和 TTS 等高成本能力；关闭后请求会在 quota 预占和上游调用前被受控拒绝。
- 邮箱注册补上项目内验证回调和重发验证邮件入口，用户可以在 `/verify-email` 重新发送验证邮件。

## 2026-05-09

### 公网注册与生成接口防护收口
- 注册页改为服务端受控注册，支持关闭注册、邀请注册和开放注册三种模式；邀请注册会校验邀请码并记录使用尝试。
- 邮箱未验证用户会停留在验证提示页，不能进入主应用或调用受保护 API。
- 练习生成、场景生成、表达生成、解释和 TTS 等高成本接口现在同时按用户和 IP 限流，并在管理员状态里展示当前限流后端。
- 高成本生成和 TTS 入口新增每日额度与调用前预占；超过当天额度时会受控拒绝，不再继续触发上游成本。
- 异常账号现在可通过账号状态限制进入主应用、生成能力或写入能力；学习时长上报也增加了单次上限和最小间隔，避免明显刷数污染个人进度。

## 2026-05-08

### Review 场景回补提交修复
- Review 页的场景回补练习现在会沿用原始练习题集的真实 `practiceSetId`，不再构造临时 ID，避免提交时被服务端题集归属校验拒绝。
- 历史异常数据缺少题集锚点时，页面会受控提示失败，不会误写入新的练习运行记录。

## 2026-05-07

### Scene / Review 下一步主路径收口
- Scene 主学习页新增“当前下一步”任务条，让用户不用打开浮动入口也能看到当前训练步骤、下一步说明和主动作。
- Scene 变体学习页把“基于此变体生成练习”保留为主动作，“删除变体”降级为辅助危险操作。
- Review 来源场景入口改为“辅助回看”分区，保留查看原场景和回到场景继续练，但不再抢占当前复习阶段的底部主 CTA。

### Review 递进式练习正式信号收口
- 普通表达复习会正式记录变体改写是否完成、选择的改写方向，以及完整输出是否用进目标表达。
- Review 调度会保守使用这些训练信号，让缺少迁移改写或目标表达完整输出的条目更容易被接回。
- Today 的 review 任务说明会优先消费服务端摘要，展示目标表达完整输出进展，而不是直接解释原始日志。

## 2026-05-06

### chunks 工作台动作层级收口
- 表达条目优先展示查看详情、开始/继续复习和来源场景回流，表达地图等整理能力收进展开态或详情上下文。
- 句子条目新增来源场景回流入口，并继续保留从句中记录表达的路径，不再把句子伪装成表达复习条目。
- chunks 详情中的“补全当前 chunk”文案改为“补全当前表达”，减少开发术语暴露。

## 2026-04-30

### scenes 循环复习后台播放更稳定
- scenes 复习播放入口改为固定顺序循环播放，会提前把一组已学场景准备成单个循环音频包，减少后台或锁屏时依赖页面切换下一段音频。
- review pack 会优先复用本地场景详情缓存和浏览器 TTS 缓存；单个候选场景加载失败时会跳过该场景，继续播放其他可用内容。
- review pack 生成或播放失败时，会回退到逐场景完整音频队列，并复用本轮已加载的场景详情，减少重复请求。
- 循环播放会显示准备状态；弱网或省流量时不再自动准备较重音频，用户点击后仍会尝试准备并播放。

## 2026-04-22

### 后台列表与 chunks 操作按钮体验收口
- 管理后台列表页改为卡片式列表，并统一顶部菜单、筛选区、下拉框和操作按钮样式。
- 后台场景、表达库、导入场景、变体、缓存和可观测性页面的主要文案收口为中文展示。
- chunks 详情里的开始复习、表达地图和移入同类组按钮修正主次层级，避免主按钮被 ghost 样式覆盖。

## 2026-04-21

### 修复场景练习页返回后的题集同步
- 从场景练习题页返回主场景页时，页面会继续后台请求服务端 latest 练习题集；本地缓存仍用于秒开，但不会阻止接口同步。
- 返回后训练入口会保持在“开始练习 / 继续整段练习”阶段，不会因为同一 scene 内部视图切换短暂回退成“听熟这段”。

### scenes 新增随机复习播放入口
- scenes 列表页新增随机复习播放入口，可以从已学习进度达到 60% 的场景中随机开始播放完整场景音频。
- 随机复习播放会按当前列表顺序继续播放后续合格场景，并复用现有 scene full TTS 缓存、失败冷却和受控提示。
- 随机播放入口与单场景完整循环播放入口统一为白底圆形按钮，播放中使用旋转圆圈状态。

### 场景练习题集改为服务端持久化
- scene 练习题现在会保存到服务端；本地缓存只负责秒开和失败降级，不再作为唯一题目来源。
- 进入同一场景时，即使本地已有缓存，页面也会后台请求服务端 latest 题集并同步覆盖，避免本地和线上题目不一致。
- 手动重新生成会创建新的题集，旧 run / attempt 保持原题集追溯；练习运行态写入前会校验题集归属。

- 生成面板同步优化了模式切换顺序、默认选中项、输入框内边距和边框样式。

## 2026-04-16

- today 页面会在学习路径顶部显示“为什么当前先做这一步”的解释，并统一 continue、repeat、review 的推荐原因与任务排序。
- 完整场景音频播放失败时会返回更稳定的中文提示，不再把上游原始错误直接暴露给页面。

## 2026-04-15

- `review submit`、`learning` 场景学习推进、`phrases save/save-all` 会对同一短窗口内的重复提交做服务端去重，降低重复点击和网络重试造成的重复写入。
- `phrases save` 和 `phrases save-all` 补上受保护写接口的来源校验，减少跨来源误调用风险。

## 2026-04-14

### 后端接口治理第一阶段基线
- 后端受保护接口会统一生成并透传 `requestId`，未知内部错误响应也会返回可追踪标识。
- 第一批高成本接口补上最小限流基线，优先保护场景导入、场景生成、练习生成、释义解释和 TTS 相关接口。
- 第一批受保护写接口补上最小同源来源校验，跨站来源不能再直接调用复习提交和学习进度这类核心写接口。

## 2026-04-07

### 场景练习体验与题目质量优化
- scene 练习题的 `cloze` 模块会按场景句子数补足填空覆盖，减少长场景只有少量填空题的情况。
- 半句复现的最小句长阈值从 6 词放宽到 5 词，模块上限从 4 题提高到 5 题。
- 练习生成失败时会优先回退到本地出题，并统一返回中文错误提示。
- 已生成的练习支持在练习页菜单里直接“重新生成题目”，不需要先删除练习再退回句子页。
- 题目页会展示 `AI生成` 或 `系统生成` 来源标记，方便区分模型生成结果和本地回退结果。
- 同一题集同一题型只会启动一次运行态，避免页面重渲染反复调用 `practice/run`。

### 生成中加载反馈统一
- 共享 loading 组件新增动态省略号能力，可统一显示“正在生成中.”、“正在生成中..”、“正在生成中...”。
- scene 练习页点击“重新生成题目”后，菜单项会切成统一加载态并禁用重复点击。
- 场景生成、导入、句子保存、后台场景动作、复习页进入场景等首批长等待入口统一到同一套加载反馈。

## 2026-04-01

### TTS 缓存、预热和播放编排收口
- 浏览器端 TTS 缓存新增内存 URL、预加载 URL、Blob URL 和 Cache Storage 上限，并在超限后按最旧条目裁剪。
- lesson 级音频预热 key 收口到共享 builder，scene detail、scene prefetch、today continue learning 等入口不再重复调度。
- 新增公共 TTS 播放控制器，统一 chunk、sentence、scene loop 的播放切换、再次点按停止、loop 状态清理和错误兜底。

### 接口边界与登录重定向安全收紧
- `/api/tts/regenerate` 只允许管理员触发，并拒绝空批量或超出上限的重生成请求。
- `explain-selection`、`practice/generate`、`scene/mutate`、`scene/parse` 等高成本接口收紧为显式受保护入口。
- `login`、`signup` 和 `middleware` 的站内重定向校验统一，危险的 `//host` 与跨站地址不再作为有效回跳目标。
- GLM 与 OpenAI 服务端调用补充超时、空响应防御和统一错误收敛。

### Review 训练信号、调度和体验增强
- `review` 普通表达复习改成 `微回忆 -> 熟悉度/输出信心 -> 变体改写 -> 完整输出 -> 结果判断` 的递进式训练流。
- `review` 最终提交会把熟悉度、输出信心和完整输出状态一起带到后端，服务端 summary 新增当天主动输出和完整输出数量。
- `review` due 列表不再只按 `next_review_at` 平铺排序，会优先前置低输出信心、未完成完整输出或仍停留在识别层的表达。
- `review` 页面补上调度提示文案，解释当前条目为什么优先出现。

### Chunks 表达删除与保存链路增强
- 删除主表达时后端会补位新主表达，删空表达簇时会返回 `clusterDeleted = true`，前端据此关闭详情或切到补位表达。
- `chunks` 页保存链路统一 `sourceNote`、`relationType`、`relationSourceUserPhraseId` 与 `expressionClusterId` 的拼装规则。
- 手动新建 chunk 时，两个底部动作按钮不再共用 loading 展示；点击“保存到表达库”时只由当前按钮显示保存中。

### Today、Scene 和学习状态语义收口
- `today` 页面继续学习入口来源、有效步骤和进度解析收口到统一 helper，减少展示语义漂移。
- `today` 和 `scene` 对练习阶段的解释统一区分“进入句子练习”“继续整段练习”和“本轮已完成”。
- 句子完成回调只在同一句第一次达到 `complete` 里程碑时触发，避免重复计入完成。
- 场景详情页不再把旧的“练核心句”作为用户侧独立前置步骤，用户可见流程收口为“听熟这段 -> 看重点表达 -> 开始练习 -> 解锁变体”。

### 类型和文案可读性修复
