# audio-action-button-consistency Specification

## Purpose
统一前端音频动作按钮的视觉形态、可访问名称，以及句子翻译/播放入口的展示规则，避免阅读层出现多套并行交互。

## Requirements
### Requirement: 音频动作按钮必须默认以纯 icon 形式展示
系统 MUST 将前端音频动作按钮统一为纯 icon 视觉形态，避免在不同页面同时存在 icon-only 与 icon+文本两套并行样式。

#### Scenario: 页面渲染朗读或循环播放入口
- **WHEN** `lesson`、`scene detail`、`chunks`、selection detail 等页面渲染音频动作按钮
- **THEN** 按钮正文不得再显示“朗读”“播放”“循环播放”等可见文本
- **AND** 按钮必须以统一 icon 样式呈现

### Requirement: 句子翻译入口必须收敛为单一层级
系统 MUST 只在句子正文下方保留翻译切换入口；进入句子详情面板或弹框后，中文翻译 MUST 直接展示，不得再保留额外一层“翻译/收起”按钮。

#### Scenario: 用户查看句子正文
- **WHEN** 用户停留在句子正文列表或卡片层
- **THEN** 系统 MAY 保留句子下方的翻译按钮
- **AND** 翻译的展开与收起仍可以在该层级完成

#### Scenario: 用户进入句子详情
- **WHEN** 用户打开句子详情面板、sheet 或等价详情容器
- **THEN** 系统必须直接展示该句中文翻译或统一 fallback
- **AND** 不得再要求用户额外点击“翻译”按钮才能看见中文

### Requirement: 句子详情播放按钮必须与句子右侧对齐
系统 MUST 把句子详情里的播放按钮固定在句子行右侧，和下方其他播放按钮保持一致的位置语言。

#### Scenario: 句子详情展示主句与动作按钮
- **WHEN** 页面渲染句子详情头部的主句和播放入口
- **THEN** 播放按钮必须位于句子右侧
- **AND** 不得再与翻译切换按钮混排
- **AND** 下方例句或相关表达的播放入口位置应继续与该右侧 icon 规则保持一致

### Requirement: 纯 icon 音频按钮必须保留可访问名称
系统 MUST 在按钮收敛为纯 icon 后继续保留明确的无障碍语义，避免视觉统一后变成不可识别的匿名按钮。

#### Scenario: 用户或测试识别音频动作按钮
- **WHEN** 用户使用辅助技术或交互测试通过语义查询音频动作按钮
- **THEN** 按钮必须保留稳定的 `aria-label` 或等价可访问名称
- **AND** 该名称必须能表达当前动作语义，例如“朗读”“停止”“循环播放”或“停止循环”

### Requirement: 音频动作按钮状态变化不得依赖可见文本
系统 MUST 允许音频动作按钮在 active 或 loading 状态下继续表达状态变化，但这种表达不得依赖重新显示正文文字。

#### Scenario: 按钮处于播放中或加载中
- **WHEN** 某个音频动作按钮进入 active 或 loading 状态
- **THEN** 按钮必须仍保持纯 icon 展示
- **AND** 状态反馈必须通过 icon 样式、动画和可访问名称完成
