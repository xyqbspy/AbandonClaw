# audio-playback-orchestration Specification

## Purpose
定义前端基于 `tts-api` 的 TTS 播放公共编排职责，统一页面播放、停止、循环、激活态与错误兜底行为，避免 `lesson`、`scene detail`、`chunks` 等入口各自维护分裂的播放状态机。

## Requirements
### Requirement: TTS 播放编排必须提供统一公共控制器
系统 MUST 为基于 `tts-api` 的前端播放入口提供统一的公共播放编排层，用于复用“播放、停止、循环、激活态、loading 态和错误兜底”这些共同行为，而不是让每个页面各自维护一套近似状态机。

#### Scenario: 页面接入公共 TTS 播放编排
- **WHEN** `lesson`、`scene detail`、`chunks` 等页面需要触发句子、chunk 或场景循环播放
- **THEN** 页面必须通过统一公共控制器消费这些通用播放动作
- **AND** 页面不得继续各自复制完整的播放/停止/循环状态判断链路

### Requirement: 再次点按同一播放目标时必须表现一致
系统 MUST 对“再次点按当前正在播放或正在加载的同一目标”提供一致行为，避免不同页面出现一处停止、一处重播的分裂体验。

#### Scenario: 再次点按当前正在播放的 chunk 或 sentence
- **WHEN** 用户再次点按与当前激活目标完全相同的 chunk 或 sentence，且该目标处于 `loading` 或 `playing` 状态
- **THEN** 公共播放编排层必须停止当前 TTS 播放
- **AND** 页面看到的激活态必须同步回到非播放状态

### Requirement: 循环播放与普通播放的前置清理必须统一
系统 MUST 在公共播放编排层统一处理播放前的停止清理与 loop 状态切换，避免不同页面在切换普通播放和循环播放时留下悬空状态。

#### Scenario: 从循环播放切换到普通播放
- **WHEN** 某页面当前处于 loop 播放中，用户触发新的普通播放目标
- **THEN** 公共播放编排层必须先停止当前播放并清理 loop 状态
- **AND** 再启动新的普通播放目标

#### Scenario: 循环播放结束或失败
- **WHEN** 循环播放链路结束、被中断或抛出异常
- **THEN** 公共播放编排层必须恢复 loop 状态
- **AND** 不得让页面残留“仍在循环中”的假状态

### Requirement: 页面特化输入必须保留在页面边界内
系统 MUST 允许页面继续提供自身业务 payload、埋点和文案，但这些页面特化逻辑不得破坏公共播放编排的一致行为。当前端把播放入口统一为纯 icon 按钮时，页面 MAY 调整按钮的视觉展示，但 MUST 继续保留稳定的无障碍名称，且不得因去掉正文文本而绕开公共播放编排层。

#### Scenario: 页面需要附带业务副作用
- **WHEN** 页面在触发公共播放前后仍需要执行埋点、预热或局部 UI 副作用
- **THEN** 页面可以保留这些业务逻辑
- **AND** 但播放、停止、激活态和错误兜底的判定必须仍由公共播放编排层统一负责

#### Scenario: 页面将播放按钮改为纯 icon
- **WHEN** 页面把朗读或循环播放入口调整为纯 icon 按钮
- **THEN** 页面必须继续通过公共播放编排层触发对应动作
- **AND** 按钮必须保留稳定的无障碍名称
- **AND** 不得因为去掉可见文本而丢失“朗读 / 停止 / 循环播放 / 停止循环 / 加载中”等状态语义
