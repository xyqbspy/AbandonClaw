## ADDED Requirements

### Requirement: Today 编排与音频可靠性改动必须同步更新维护说明
当变更涉及 `today` 编排规则、TTS 降级行为或批量音频维护能力时，维护流程 MUST 同步更新相应说明文档与回归点，而不能只停留在实现代码。

#### Scenario: 修改 today 首要任务规则
- **WHEN** 维护者调整 `today` 首要任务排序或解释逻辑
- **THEN** 必须同步更新对应说明文档
- **AND** 必须补充或更新最小回归测试点

#### Scenario: 修改 TTS 降级或批量重生成行为
- **WHEN** 维护者调整 TTS 失败降级、预热或批量重生成逻辑
- **THEN** 必须同步更新音频链路说明文档
- **AND** 必须记录相关验证方式与剩余风险

### Requirement: 发布前检查必须覆盖安全头与音频异常收口
维护流程 MUST 在发布前检查中覆盖基础安全头、TTS 上游异常样本和剩余后台白名单入口，确保生产化收口不是口头约定。

#### Scenario: 维护者执行上线前检查
- **WHEN** 维护者执行发布前 checklist
- **THEN** 检查项 MUST 包含安全头配置、TTS 异常复核和剩余白名单入口确认
- **AND** 不得只记录成功路径而忽略异常样本与例外入口

### Requirement: 学习反馈与业务可观测性改动必须同步更新维护说明
当变更涉及学习完成反馈、review 结果反馈、关键业务事件或音频失败替代体验时，维护流程 MUST 同步更新说明文档、验证命令与剩余风险记录，而不能只停留在实现代码。

#### Scenario: 修改学习反馈或业务事件记录
- **WHEN** 维护者调整 today / review / lesson complete 的反馈展示或关键业务事件记录
- **THEN** 必须同步更新对应说明文档
- **AND** 必须补充或更新最小回归测试点与验证记录

### Requirement: 发布前检查必须覆盖业务失败摘要与音频替代入口
维护流程 MUST 在发布前检查中覆盖业务级失败摘要是否可追踪、音频失败替代 CTA 是否可用，以及结果反馈是否仍然遵守学习证据边界。

#### Scenario: 维护者执行上线前检查
- **WHEN** 维护者执行发布前 checklist
- **THEN** 检查项 MUST 包含业务失败摘要、音频替代入口和学习反馈边界确认
- **AND** 不得只检查请求成功率而忽略业务层失败路径
