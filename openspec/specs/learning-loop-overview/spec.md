## Purpose

定义项目当前稳定的学习产品闭环，确保后续新增功能时，不会破坏 `today`、`scene`、`chunks`、`review` 与学习状态同步之间的关系。
## Requirements
### Requirement: Today 必须作为每日学习入口
系统 MUST 提供 `today` 入口，把继续学习、复习任务和学习概览聚合成当日可执行入口。

#### Scenario: 用户进入今日页
- **WHEN** 用户打开 `today`
- **THEN** 页面应展示继续学习入口、复习相关信息或学习概览中的至少一类核心信息
- **AND** 这些信息应来自学习链路的聚合结果，而不是孤立静态展示

### Requirement: Scene 必须作为主学习流程工作台
系统 MUST 允许用户在 `scene` 中完成语境阅读、句子交互、表达提取、练习推进与变体扩展等核心学习行为，并保证这些用户可见状态不会因为本地缓存命中而长期停留在过期结果。系统 MUST 同时把场景步骤推进、句子里程碑升级、场景练习完成与场景 done 视为不同层级的学习结果，并在用户可见链路中保持一致解释。

#### Scenario: 用户在场景中学习
- **WHEN** 用户进入某个 `scene`
- **THEN** 可以围绕句子、chunk、练习、变体或表达地图进行学习推进
- **AND** 场景交互应与学习状态同步链路保持一致
- **AND** 用户可见的场景状态、进度和关联入口在命中本地缓存后仍必须继续向服务端校验最新结果
- **AND** 用户从进入句子练习、推进句子里程碑、完成场景练习到进入场景 done 的解释必须前后一致

### Requirement: Scene 显式步骤必须与当前产品链路一致
系统 MUST 让用户在场景内看到与当前产品一致的显式步骤，不得继续暴露已经并入练习流程的旧前置步骤。

#### Scenario: 用户查看场景训练步骤
- **WHEN** 用户打开场景详情页中的训练浮层、继续学习卡片或其他用户可见学习路径入口
- **THEN** 显式步骤应围绕“听熟这段 -> 看重点表达 -> 开始练习 -> 解锁变体”组织
- **AND** 已兼容保留的 `practice_sentence` / `sentence_practice` 状态只能作为内部里程碑或兼容状态存在
- **AND** 用户侧不得再被要求执行独立的“练核心句”前置步骤

### Requirement: Chunks 必须承担表达沉淀与整理职责
系统 MUST 允许用户把来自场景或手动录入的表达沉淀为可管理资产，并支持后续扩展、聚类、对比或复习组织，同时保证列表内容可以先展示缓存再继续刷新，不因缓存命中而长期隐藏服务端最新变化。

#### Scenario: 用户保存并整理表达
- **WHEN** 用户把一个表达加入个人资产
- **THEN** 该表达后续应能在 `chunks` 或相关详情流中继续被查看、补全、组织或加入复习
- **AND** `chunks` 列表在命中本地缓存时仍必须继续执行后台刷新
- **AND** 若服务端结果已变化，页面与本地缓存都必须更新到最新状态

### Requirement: 学习行为必须可被服务端学习链路消费
系统 MUST 将关键学习行为映射为可持续追踪的学习状态，而不是只停留在前端瞬时 UI。

#### Scenario: 用户开始、推进或结束学习
- **WHEN** 用户进入学习、停留推进、完成或离开学习流程
- **THEN** 系统应能把这些行为纳入学习状态同步或统计链路
- **AND** `today`、`review`、`progress` 等聚合页面可以消费这些结果
- **AND** “进入句子练习”“句子达到 complete 里程碑”“场景练习完成”“场景 done”必须作为可区分的学习结果被稳定消费
