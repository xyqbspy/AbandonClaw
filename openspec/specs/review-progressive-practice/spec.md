## Purpose
定义 `review` 页面从识别到输出的递进式练习层级，确保单次复习流程能够按掌握状态提供分层训练路径，而不是退回到单层反馈流程。该 capability 在学习闭环中承接 `review` 的阶段式工作台与训练路径设计，而不重复定义正式后端信号、调度或来源契约。
## Requirements
### Requirement: Review 递进式练习必须覆盖识别到输出的能力层级
系统 MUST 为单次 `review` 提供从识别到输出的递进式练习层级，至少覆盖熟悉度/输出信心、微回忆、变体改写和完整输出四类训练信号。

#### Scenario: 系统为普通表达生成单次复习流程
- **WHEN** 一条普通表达进入 `review`
- **THEN** 页面必须能把本次复习映射到识别、主动提取、迁移改写或完整输出中的一个或多个阶段
- **AND** 不得再只把单次复习压缩成“看参考后直接 again/hard/good”的单层流程

### Requirement: Review 递进式练习必须允许分层路径
系统 MUST 根据当前复习项类型、掌握状态或训练目标，决定本次复习是否需要执行完整输出，而不是要求所有复习项都走满全部阶段。系统 MUST 同时保证调度节奏与这些训练层级保持一致，避免页面已区分识别级别与输出级别，但调度仍对不同层级无差别处理。

#### Scenario: 系统渲染不同难度的复习项
- **WHEN** 页面为不同掌握度的复习项构建训练流程
- **THEN** 必须允许较轻复习只执行微回忆与判断
- **AND** 必须允许较重复习继续进入变体改写或完整输出
- **AND** 训练深度必须来自稳定映射规则，而不是由页面临时散落判断
- **AND** 若条目仍停留在较低训练层级，系统后续调度不得把它与已完成完整输出的条目视作同等节奏

### Requirement: Review 递进式练习阶段展示必须遵守正式信号边界
系统 MUST 在页面阶段展示中明确区分哪些递进式练习阶段已经接通正式后端记录，哪些仍只是前端 TODO 占位，不得把未接通阶段伪造成正式学习状态或调度信号。涉及正式后端信号、与最终反馈并存关系和聚合消费的稳定语义，MUST 以 `review-practice-signals` capability 为准，而不是在页面流程规范中重复定义。

#### Scenario: 某个递进式阶段尚未接通后端
- **WHEN** 页面展示熟悉度、自信度、变体改写或完整输出中的某个阶段
- **THEN** 若该阶段尚未有正式后端契约，前端必须明确标记其为待接入能力
- **AND** 该阶段结果不得被直接当作正式学习完成、review 调度或长期统计结果
- **AND** 若某个阶段已经升级为正式信号，其正式语义必须遵守 `review-practice-signals` 中定义的并存与聚合规则

### Requirement: Review 递进式练习必须提供失败降级路径
系统 MUST 在用户无法完成更高强度输出时，提供回退到更低提示强度训练的稳定路径，而不是只给出一次性失败判断。

#### Scenario: 用户在高强度输出阶段失败
- **WHEN** 用户在变体改写或完整输出阶段无法完成当前任务
- **THEN** 系统必须允许其回退到微回忆、参考提示或回到原场景继续练习
- **AND** 页面必须把这种回退解释为训练降级，而不是直接结束本次复习

### Requirement: Review 变体改写阶段正式化后必须保持训练降级边界
系统 MUST 在普通表达 review 的变体改写阶段记录用户是否完成固定方向改写，但不得把未完成改写直接等同于本次 review 失败。变体改写正式信号只能说明用户是否进入迁移训练，最终队列推进仍 MUST 由最终 feedback 和正式调度规则共同决定。

#### Scenario: 用户完成变体改写后提交 review
- **WHEN** 用户在普通表达 review 中选择固定改写方向并提交非空改写草稿
- **THEN** 页面提交 MUST 带上变体改写完成状态和对应固定方向 id
- **AND** 后端 MUST 能把该状态记录为本次 review 事件的正式信号
- **AND** 页面不得要求该改写通过 AI 评分后才能继续最终 feedback

#### Scenario: 用户跳过或未完成变体改写
- **WHEN** 用户未完成变体改写但继续走到最终 feedback
- **THEN** 系统 MUST 把该状态解释为迁移训练未完成
- **AND** 不得直接把它解释为本次 review 失败
- **AND** 页面 MUST 继续提供参考、回退或最终反馈入口

### Requirement: Review 完整输出阶段必须区分完成状态与目标表达覆盖
系统 MUST 在完整输出阶段继续记录用户是否完成输出，并额外区分完整输出是否确定性覆盖目标表达。页面 MUST 把目标表达覆盖解释为输出训练信号，而不是语法、自然度或完整质量评分。

#### Scenario: 用户完整输出包含目标表达
- **WHEN** 用户提交的完整输出经过确定性归一化后包含目标表达
- **THEN** 系统 MUST 记录完整输出已完成
- **AND** 系统 MUST 记录目标表达覆盖为已命中
- **AND** 后续页面说明 MUST 避免把该结果称为 AI 质量评分

#### Scenario: 用户完整输出未覆盖目标表达
- **WHEN** 用户提交了非空完整输出但未确定性覆盖目标表达
- **THEN** 系统 MUST 记录完整输出已完成
- **AND** 系统 MUST 记录目标表达覆盖为未命中
- **AND** 页面 MUST 将其解释为“本轮还没把目标表达用进完整输出”的训练提示

#### Scenario: 用户未填写完整输出
- **WHEN** 用户未填写完整输出就提交最终 feedback
- **THEN** 系统 MUST 记录完整输出未开始
- **AND** 系统 MUST 记录目标表达覆盖为未开始
