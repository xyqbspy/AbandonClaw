## Purpose

定义学习主链路中主按钮与次按钮的统一层级规则，确保 `scene / lesson / chunks / scenes / review / today` 之间同语义动作的视觉表达稳定一致，降低样式漂移和后续维护歧义。

## Requirements

### Requirement: 学习主链路中的主动作按钮必须使用统一主按钮层级
系统 MUST 在学习主链路中把推进当前学习步骤的唯一主要动作呈现为统一主按钮样式，避免同类主动作在不同页面分别表现为白底按钮、浅色按钮或局部特例样式。

#### Scenario: 用户查看学习页面中的主要推进动作
- **WHEN** 用户在 `scenes`、`scene detail`、`lesson detail`、`chunks detail`、`review` 或 `today` 中看到会直接推进当前学习步骤的主要动作
- **THEN** 该按钮必须使用统一主按钮视觉层级
- **AND** 不得因为页面归属不同而退回为次按钮或局部自定义底色

### Requirement: 学习主链路中的保存类动作必须使用统一次按钮层级
系统 MUST 在学习主链路中把“保存到表达库 / 保存并加入复习 / 保存句子 / 添加到表达库”等保存、沉淀、入库类动作呈现为统一次按钮样式，确保颜色、边框、底色和文字层级保持一致。

#### Scenario: 用户查看学习页面中的保存与入库动作
- **WHEN** 用户在 `lesson`、`chunks` 或其它学习主链路页面中看到保存类动作
- **THEN** 这些按钮必须使用统一次按钮视觉层级
- **AND** 不得继续混用白底、浅色底、无边框局部按钮或页面特例样式

### Requirement: 同一动作语义不得因页面不同改变主次层级
系统 MUST 以动作语义而不是页面归属决定按钮层级；同样属于“开始复习 / 加入复习 / 生成场景”这类主动作的按钮，必须跨页面保持一致，同样属于“保存到表达库 / 保存句子”这类次动作的按钮也必须跨页面保持一致。

#### Scenario: 用户在不同学习页面之间切换
- **WHEN** 用户从 `scene`、`lesson`、`chunks` 或 `scenes` 等页面切换并遇到相同语义的动作按钮
- **THEN** 相同语义的按钮必须保持一致的主次层级
- **AND** 维护者不得仅因某个页面已有局部样式就改变该动作的语义层级
