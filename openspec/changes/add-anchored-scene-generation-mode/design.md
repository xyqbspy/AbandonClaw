# 设计说明：句子锚点场景生成

## Status
draft

## Current Flow
当前入口：
- `scenes` 页通过 `GenerateSceneSheet` 打开“生成场景”底部面板

当前处理链路：
- 用户输入 `promptText`
- 前端调用 `/api/scenes/generate`
- request schema 只接收 `promptText/tone/difficulty/sentenceCount/reuseKnownChunks`
- 服务端把 `promptText` 视为“用户想练的情境”
- prompt 要求模型生成一个短英文对话场景
- 结果经 parse/import 后落成 imported scene

当前回写：
- 场景创建成功后刷新 scenes 列表

当前回退路径：
- 输入为空或过短时前端报错
- API/模型失败时走统一生成失败提示

## Problem
当前链路把所有输入都压成一个 `promptText`，导致两类语义被混在一起：

1. 用户描述一个情境  
2. 用户给出一句想保留的英文

这会带来三个问题：

- 入口语义不清：用户不知道系统会把这句英文当“提示词”还是“必须保留的核心句”
- 服务端约束缺失：模型没有被要求围绕该句组织冲突、关系和上下文
- 测试不可判定：现有测试很难判断什么叫“支持按句子生成场景”

## Stability Closure
### In This Round
- 明确“同一入口、双模式”的产品边界
- 明确请求契约里的模式字段，停止只靠自由文本猜语义
- 明确 `anchor_sentence` 模式下最小结果保证，避免实现与预期漂移
- 明确不新增第二入口，继续复用 scenes 页现有主链路

### Not In This Round
- 不做锚点句扩展属性持久化
- 不做锚点句生成后的学习资产自动沉淀
- 不做多候选场景对比与挑选

## Decision
### 1. 入口保持在现有生成场景 sheet 内
理由：
- 用户心智连续，仍然是“我给输入，系统生成场景”
- 可以复用现有 scenes 页和生成刷新链路
- 避免新增页面、入口分叉和状态维护成本

### 2. UI 使用明确的模式切换，而不是只改 placeholder
建议模式：
- `按情境生成`
- `按句子生成`

理由：
- 两种输入都叫“prompt”，但生成约束不同
- 明示模式比隐式猜测更容易测试、更容易维护

### 3. 请求契约增加 `mode`
新增字段建议：
- `mode: "context" | "anchor_sentence"`
- 保留 `promptText`
- 可选增加 `contextHint`，仅在 `anchor_sentence` 模式下用于补充语气/关系/场景

最小方案优先级：
- 第一阶段至少先加 `mode`
- `contextHint` 可作为同轮可选增强，但不是必须

### 4. 服务端 prompt 按模式分流
`context` 模式：
- 保持现有“根据情境生成短场景”的主语义

`anchor_sentence` 模式：
- 明确声明输入是一句必须围绕其构建场景的核心句
- 场景必须自然包含该句
- 该句应成为对话中的关键表达，而不是边缘出现一次
- 仍要满足总句数、口语自然度和可练习性约束

### 5. 向后兼容策略
- 老请求没有 `mode` 时，默认按 `context`
- 这样不会破坏现有场景生成调用方

### 6. 测试基线围绕“模式语义”建立
最小必须覆盖：
- UI 切换后输入文案变化
- request schema 能识别并规范化 `mode`
- `anchor_sentence` 模式会把锚点约束传入 prompt / service
- 生成结果校验至少覆盖“锚点句必须出现在输出对话文本中”

## Risks
- 风险 1：如果仅检查“文本中包含原句”，仍可能出现“机械硬塞一句”的低质量场景
  - 处理：本轮先保证存在性和核心性约束，质量优化延后
- 风险 2：如果把锚点句改写自由度放太大，用户会觉得“这句没被真正练到”
  - 处理：本轮默认要求原句保留；变体策略如需开放，必须显式定义
- 风险 3：增加 `contextHint` 会让 UI 稍复杂
  - 处理：先把它设计为可选增强，不阻塞主提案

风险去向：
- 本 change 的 tasks 和后续实现验证记录

## Validation
验证方式：
- scenes 页交互测试
- request schema / prompt builder 单测
- 服务端生成链路最小回归测试

回归范围：
- 现有 `context` 模式不回归
- Scenes 页生成成功后的刷新链路不回归
- 生成缓存 key 在模式变化时能区分

未覆盖风险：
- 模型层面对“核心性”的主观质量仍有波动
- 中文输入锚点句与英文输入锚点句的混合场景需要实现时再细化

本轮已收口的不稳定点：
- 场景生成入口语义不再只靠自由文本承载
- “句子生成场景”有正式契约可依

明确延后的不稳定点：
- 多变体生成
- 台词来源元数据
- 与表达保存/review 的联动沉淀
