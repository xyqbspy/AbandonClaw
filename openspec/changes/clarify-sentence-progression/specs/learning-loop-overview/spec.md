## MODIFIED Requirements

### Requirement: Scene 必须作为主学习流程工作台
系统 MUST 允许用户在 `scene` 中完成语境阅读、句子交互、表达提取、练习推进与变体扩展等核心学习行为，并保证这些用户可见状态不会因为本地缓存命中而长期停留在过期结果。系统 MUST 同时把场景步骤推进、句子里程碑升级、场景练习完成与场景 done 视为不同层级的学习结果，并在用户可见链路中保持一致解释。

#### Scenario: 用户在场景中学习
- **WHEN** 用户进入某个 `scene`
- **THEN** 可以围绕句子、chunk、练习、变体或表达地图进行学习推进
- **AND** 场景交互应与学习状态同步链路保持一致
- **AND** 用户可见的场景状态、进度和关联入口在命中本地缓存后仍必须继续向服务端校验最新结果
- **AND** 用户从进入句子练习、推进句子里程碑、完成场景练习到进入场景 done 的解释必须前后一致

### Requirement: 学习行为必须可被服务端学习链路消费
系统 MUST 将关键学习行为映射为可持续追踪的学习状态，而不是只停留在前端瞬时 UI。

#### Scenario: 用户开始、推进或结束学习
- **WHEN** 用户进入学习、停留推进、完成或离开学习流程
- **THEN** 系统应能把这些行为纳入学习状态同步或统计链路
- **AND** `today`、`review`、`progress` 等聚合页面可以消费这些结果
- **AND** “进入句子练习”“句子达到 complete 里程碑”“场景练习完成”“场景 done”必须作为可区分的学习结果被稳定消费
