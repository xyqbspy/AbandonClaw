## Context

当前场景学习链路已经包含三层互相耦合的状态：

- 场景显式步骤：`listen -> focus_expression -> practice_sentence / scene_practice -> done`
- 练习内部题型：`cloze -> guided_recall -> sentence_recall -> full_dictation`
- 句子里程碑：`keyword -> structure -> complete`

这些状态目前分别散落在 `scene-detail-selectors`、`scene-detail-page`、`scene-practice-view`、`today-page-selectors` 和若干共享文案文件中。现状能工作，但“进入练习”和“完成练习”的边界、句子级完成与场景级完成的关系、以及 `today`/`scene` 对同一状态的展示口径还没有被一份稳定规范统一下来。

## Goals / Non-Goals

**Goals:**

- 明确场景步骤、题型步骤、句子里程碑三层状态的职责边界。
- 定义用户进入句子后的推进顺序，以及每一层状态的完成条件。
- 明确“完成当前句子练习”和“完成当前场景练习”不是同一事件，并写清它们之间的升级关系。
- 约束 `today`、场景训练浮层、场景练习页对完成状态的消费口径，保证同一学习状态不会出现不同解释。
- 为后续 implementation 提供可直接落地的测试回归基线。

**Non-Goals:**

- 本次不重做练习 UI 视觉结构。
- 本次不扩展新的练习题型，也不调整 AI 出题策略。
- 本次不改 `review` 独立复习模式的排序、推荐或算法。
- 本次不承诺修改数据库模型；若现有字段足够表达，则优先复用。

## Decisions

### 1. 把状态拆成三层规范，而不是继续沿用“一个 currentStep 兜底全部语义”

原因：
- `currentStep` 适合表达场景工作台当前推荐动作，不适合直接承担句子掌握程度和练习模块完成度。
- 句子里程碑是跨题型聚合结果，题型完成是模块内结果，两者不能互相替代。

备选方案：
- 继续只在页面实现里补条件分支。
  不采用，因为这会继续放大 `scene-detail-page.tsx` 和 `today-page-selectors.ts` 的隐式耦合。

### 2. 句子完成条件以“句子达到 complete 里程碑”为准，而不是“用户做过一次练习”或“进入练习页”

原因：
- 现有 `practicedSentenceCount >= 1` 只能证明用户已经开始句子练习，不能证明当前句子已完整复现。
- 如果把“进入练习”误当成“完成句子”，会导致场景步骤推进过快，训练浮层和总结态失真。

备选方案：
- 以单个 exercise 正确一次作为句子完成。
  不采用，因为句子掌握跨多个题型和多次评估升级，更适合以里程碑的最高等级来判定。

### 3. 场景练习完成条件以“当前 practice set 所有必做题型完成”为准，场景 done 以“练习完成且下游 unlock 条件满足”为准

原因：
- 题型完成是练习模块层的闭环；`done` 更接近场景工作台层面的“本轮场景已收束，可以去变体/后续环节”。
- 这样可以避免 `scene_practice` 和 `done` 在页面上被混成同一状态。

备选方案：
- 练习完成后直接判场景 done。
  不采用，因为当前产品还有变体解锁和下游入口，`done` 需要保留更高一级的语义。

### 4. `today` 与场景页必须消费同一套状态解释，不允许一个看“进入练习”，另一个看“练习完成”

原因：
- `today` 是学习入口，如果它把“开始练习”误判成“已完成场景输入”，用户会在跨页面时感到流程跳变。
- 统一口径后，才能稳定维护聚合提示、按钮文案和继续学习入口。

备选方案：
- 允许聚合页做自己的简化映射。
  不采用，因为这会让链路真相分裂。

## Risks / Trade-offs

- [风险] 现有前后端字段可能只记录“开始练习”，不足以表达“句子 complete 里程碑”。
  -> 先在 proposal 阶段把语义分清，implementation 阶段再决定是复用现有快照聚合，还是补更明确的记录字段。

- [风险] 场景训练浮层、练习页和 `today` 会同时受影响，回归面较大。
  -> 在 tasks 中单独列出 selector、页面编排和交互测试，避免只改一处。

- [风险] 用户已存在的旧学习状态可能沿用兼容字段，例如 `practice_sentence` / `sentence_practice`。
  -> 保留兼容读取，但明确它们只能表示阶段位置，不能直接等同于句子掌握完成。
