## MODIFIED Requirements

### Requirement: 高成本接口必须有显式访问边界

任何会触发外部模型调用、批量重生成、重型解析或其他高成本服务端处理的接口 MUST 具备与成本等级匹配的显式访问边界。默认情况下，这类接口 MUST 要求已登录用户访问；若接口具备批量重生成、后台维护或其他敏感运维属性，系统 MUST 进一步限制为管理员能力，而不能仅依赖“已登录即可调用”。

#### Scenario: 普通登录用户访问管理员级高成本接口

- **WHEN** 普通登录用户请求批量音频重生成或其他管理员级高成本接口
- **THEN** 系统 MUST 在进入实际重生成或重型处理前直接拒绝请求
- **AND** 不得继续删除旧资源、触发外部服务或执行高成本任务

#### Scenario: 管理员访问管理员级高成本接口

- **WHEN** 管理员请求批量音频重生成或其他管理员级高成本接口
- **THEN** 系统 MUST 允许请求进入后续业务处理
- **AND** 后续 route 或 service 仍 MUST 保留参数校验与业务约束

### Requirement: 模型与重操作接口必须在入口限制输入规模

任何会触发模型调用、批量处理或高成本服务端任务的接口 MUST 在进入核心业务逻辑前完成输入规模校验。系统 MUST 对关键文本字段长度、数组项数、批量数量或结构规模设置明确上限，并在超限时返回受控校验错误，而不能把异常大请求直接交给下游模型、文件系统或存储服务。

#### Scenario: 高成本接口收到超限输入

- **WHEN** 客户端向高成本接口提交超出文本长度、数组数量或结构规模上限的请求
- **THEN** 系统 MUST 在入口层返回校验失败响应
- **AND** 不得继续发起模型调用、文件删除、资源上传或其他高成本处理

#### Scenario: 高成本接口收到合法范围输入

- **WHEN** 客户端向高成本接口提交满足字段与规模约束的请求
- **THEN** 系统 MUST 允许请求进入后续业务处理
- **AND** 这些约束 MUST 在测试中被显式覆盖

### Requirement: 登录后热路径不得重复执行可避免的认证查询

登录成功后的用户资料热路径 MUST 复用单次身份识别结果驱动后续资料读取。系统 MUST 避免在同一请求内重复执行可合并的 session、user 或 profile 认证查询。

#### Scenario: 客户端请求当前用户资料

- **WHEN** 客户端调用 `/api/me` 请求当前用户资料
- **THEN** 系统 MUST 以单次身份识别结果驱动 profile 读取与响应构建
- **AND** 不得在同一请求内重复执行可避免的认证查询

### Requirement: 未知服务端错误必须对外收敛

所有 API route MUST 区分“已定义应用错误”和“未知内部错误”。系统 MAY 向客户端返回显式定义过的认证、鉴权或校验错误，但对于未知内部错误 MUST 返回受控的通用失败响应，而不能直接暴露底层依赖、数据库、文件系统或上游供应商返回的原始错误文本。

#### Scenario: route 抛出未知内部错误

- **WHEN** route 或 service 抛出未映射为显式应用错误的内部异常
- **THEN** 系统 MUST 记录可诊断日志并返回受控的通用失败响应
- **AND** 不得把内部错误 message 原样暴露给客户端
