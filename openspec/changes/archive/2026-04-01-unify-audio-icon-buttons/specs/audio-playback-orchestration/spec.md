## MODIFIED Requirements

### Requirement: 页面特化输入必须保留在页面边界内
系统 MUST 允许页面继续提供自身业务 payload、埋点和文案，但这些页面特化逻辑不得破坏公共播放编排的一致行为。当前端把播放入口统一为纯 icon 按钮时，页面 MAY 调整按钮的视觉展示，但 MUST 继续保留稳定的无障碍名称，且不得因去掉正文文本而绕开公共播放编排层。

#### Scenario: 页面需要附带业务副作用
- **WHEN** 页面在触发公共播放前后仍需要执行埋点、预热或局部 UI 副作用
- **THEN** 页面可以保留这些业务逻辑
- **AND** 但播放、停止、激活态和错误兜底的判定必须仍由公共播放编排层统一负责

#### Scenario: 页面将播放按钮改为纯 icon
- **WHEN** 页面把朗读或循环播放入口调整为纯 icon 按钮
- **THEN** 页面必须继续通过公共播放编排层触发对应动作
- **AND** 按钮必须保留稳定的无障碍名称
- **AND** 不得因为去掉可见文本而丢失“朗读 / 停止 / 循环播放 / 停止循环 / 加载中”等状态语义
