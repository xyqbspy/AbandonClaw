## Context

当前仓库里音频动作已经有公共组件 `TtsActionButton` 与 `LoopActionButton`，但默认行为仍允许按钮在 icon 外再渲染“朗读”“停止”“循环播放”“加载中...”等正文文本。翻译入口也存在重复展开：句子卡片下方有翻译按钮，selection detail / sentence detail 里仍保留另一层“翻译”切换按钮，导致同一中文内容需要用户重复点开。

结果是在实际页面里形成了多种形态：

- 有的入口是纯 icon
- 有的入口是 icon + 文本
- 有的和翻译、收藏按钮并排时高度与宽度不一致
- 有的测试依赖可见文本“朗读”，有的依赖 `aria-label`
- 有的句子详情直接展示中文，有的还要额外点击“翻译”

这类差异主要集中在：

- `lesson-reader` 及其 detail / toolbar 相关组件
- `sentence-block`
- `selection-detail-primitives`、`selection-detail-panel`、`selection-detail-sheet`
- `chunks` 列表、focus detail、例句卡片
- 若干 selection detail 组件与对应交互测试

项目刚刚把播放编排逻辑统一到公共 controller，按钮视觉层如果继续分裂，会让“底层动作一致、表层入口不一致”的问题继续存在。

## Goals / Non-Goals

**Goals:**

- 统一前端音频动作按钮为纯 icon 视觉形态
- 保留稳定的无障碍语义，确保按钮仍能通过 `aria-label` 被识别和测试
- 统一常见按钮尺寸与 spacing，减少播放按钮在不同页面的视觉分裂
- 明确“只有句子下方保留翻译按钮，其余详情直接展示中文翻译”的统一规则
- 把句子详情里的播放入口固定到句子行右侧，与下方其它播放按钮保持一致位置语言
- 补齐受影响交互测试，避免继续依赖可见文本“朗读”

**Non-Goals:**

- 不改 TTS 播放逻辑、缓存、预热和 controller 行为
- 不重做整页布局，只调整音频动作入口的展示方式
- 不把所有按钮组件都统一，只覆盖音频相关动作按钮与句子翻译入口

## Decisions

### 1. 公共音频按钮默认改为 icon-only

决策：`TtsActionButton` 和 `LoopActionButton` 默认走纯 icon 展示，正文文字不再默认渲染；语义通过 `aria-label`、`title` 或稳定测试钩子保留。

原因：

- 现在的问题不是缺少 icon-only 能力，而是默认允许文本扩散到各页面
- 从默认值收紧，比让每个页面手工传 `iconOnly` 更容易长期一致

备选方案：

- 方案 A：保留当前默认，逐页手工改成 `iconOnly`
  结果：容易遗漏，后续新增入口还会继续长出文本按钮

### 2. 可访问名称保留，不把“纯 icon”做成“无文本语义”

决策：所有纯 icon 音频按钮仍必须保留 `aria-label`，根据状态表达“朗读 / 停止 / 循环播放 / 停止循环 / 加载中”等语义。

原因：

- 这次要统一的是视觉，不是牺牲可访问性
- 现有交互测试大多已经能消费 `getByRole(..., { name })`，只需要把依赖可见文字的断言迁移到无障碍名称

### 3. 只保留句子下方翻译按钮，详情里直接显示中文

决策：句子正文下面现有的翻译按钮保留；但一旦进入句子详情面板或弹框，不再保留额外的“翻译/收起”切换按钮，而是默认直接展示中文翻译。

原因：

- 详情层本来就是为了展开更多信息，再让用户额外点一次“翻译”属于重复交互
- 用户刚进入详情时最常看的就是句子中文，直接显示更符合当前阅读路径

备选方案：

- 方案 A：所有地方都保留翻译切换按钮
  结果：交互层级冗余，详情里会继续出现“点开详情后再点开翻译”的重复动作

### 4. 句子详情里的播放按钮固定在句子右侧

决策：句子详情头部的播放按钮与正文句子放在同一行右侧，下方其他例句/表达播放入口继续沿用右侧 icon 位置，形成一致的阅读扫描路径。

原因：

- 现在有些详情组件里播放按钮和翻译切换混在同一行，位置语言不稳定
- 右侧单独放播放 icon，最容易和下方一整列例句播放入口对齐

### 5. 先统一公共音频按钮，不在页面里额外造新的 icon 变体

决策：页面继续复用 `TtsActionButton` / `LoopActionButton`，不再为 `lesson`、`chunks` 单独造一批专用 icon 按钮。

原因：

- 这类组件本来就已经是公共入口
- 再分叉一层只会把“统一样式”做成新的重复实现

## Risks / Trade-offs

- [测试回归] → 把交互测试从“依赖可见文本”迁到 `aria-label` 或稳定选择器，避免按钮去文案后测试误红
- [局部布局空白变化] → 优先复用现有 `size="icon"` / `size="icon-sm"` 和统一 class，必要时只做最小 spacing 调整
- [状态语义丢失] → 保持 active/loading 状态时的 `aria-label` 仍随状态变化，避免 icon-only 后无法表达“停止”或“加载中”
- [详情里直接展示中文后信息密度上升] → 只移除重复的翻译切换层，不新增额外段落；缺翻译时继续走统一 fallback
