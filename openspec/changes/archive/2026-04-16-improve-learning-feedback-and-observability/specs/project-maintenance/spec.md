## ADDED Requirements

### Requirement: 学习反馈与业务可观测性改动必须同步更新维护说明
当变更涉及学习完成反馈、review 结果反馈、关键业务事件或音频失败替代体验时，维护流程 MUST 同步更新说明文档、验证命令与剩余风险记录，而不能只停留在实现代码。

#### Scenario: 修改学习反馈或业务事件记录
- **WHEN** 维护者调整 today / review / lesson complete 的反馈展示或关键业务事件记录
- **THEN** 必须同步更新对应说明文档
- **AND** 必须补充或更新最小回归测试点与验证记录

### Requirement: 发布前检查必须覆盖业务失败摘要与音频替代入口
维护流程 MUST 在发布前检查中覆盖业务级失败摘要是否可追踪、音频失败替代 CTA 是否可用，以及结果反馈是否仍然遵守学习证据边界。

#### Scenario: 维护者执行上线前检查
- **WHEN** 维护者执行发布前 checklist
- **THEN** 检查项 MUST 包含业务失败摘要、音频替代入口和学习反馈边界确认
- **AND** 不得只检查请求成功率而忽略业务层失败路径
