## Context

当前匿名试用能力已经具备基础治理：`ALLOW_ANONYMOUS_TRIAL`、匿名身份、匿名配额、分享场景只读页、预生成 TTS 播放和注册引导。但产品入口仍然是单场景，无法覆盖“浏览多个场景 -> 选择感兴趣场景 -> 听读 -> 看句子细节 -> 尝试练习 -> 遇到写入边界后注册”的完整转化路径。

本设计选择新增独立 `/trial` 体验层，而不是直接开放主应用 `/scenes`、`/scene/[slug]`。这样可以复用匿名治理基础，同时保持主应用登录态语义干净，降低把匿名分支散落到已登录主链路的风险。

## Goals / Non-Goals

**Goals:**

- 匿名访客可以在 `/trial` 看到 4-5 条精选公开场景。
- 匿名访客可以进入 `/trial/scene/[slug]` 查看场景详情、句子、chunk/detail 信息，并播放已生成 TTS。
- 匿名访客可以查看并本地作答预生成练习题，但不能提交结果。
- 匿名访客触发导入、生成、保存、加入复习、提交、写 progress/review 等动作时，获得清晰注册提示。
- 所有匿名试用能力继续受 `ALLOW_ANONYMOUS_TRIAL` 和现有匿名配额保护。

**Non-Goals:**

- 不开放主应用路由作为匿名学习主链路。
- 不新增匿名 AI 生成能力或提高匿名配额。
- 不持久化匿名练习答案、完成状态或复习状态。
- 不把匿名 session 数据迁移到注册账号。
- 不重构已登录用户场景详情页或练习主流程。

## Decisions

### 1. 新增 `/trial` 独立试用层

`/trial` 负责展示精选公开场景列表，`/trial/scene/[slug]` 负责匿名场景详情。中间件只对白名单路径放行匿名访问：

- `/trial`
- `/trial/scene/[slug]`
- 既有 `/share/scene/[slug]`
- 既有 `/api/anonymous/*`

主应用路由 `/scenes`、`/scene/[slug]`、`/today`、`/review`、`/chunks`、`/progress` 仍要求登录。

### 2. 精选场景采用 allowlist

MVP 使用固定 slug allowlist 选择 4-5 条公开场景，避免匿名列表被用户数据、排序策略或未审核内容影响。allowlist 中的场景必须满足：

- 场景本身可公开读取。
- 句子和详情数据已生成。
- 主要句子已有可播放 TTS。
- 若展示练习入口，则预生成练习题已存在。

后续可以把 allowlist 迁移为后台配置，但不作为本轮范围。

### 3. 详情页复用匿名组件，不复用强登录组件

匿名详情页应复用或扩展 `src/features/anonymous-trial/` 下的只读组件，而不是直接复用强依赖已登录 API 的主场景详情组件。原因是主场景详情组件通常包含 progress、保存表达、练习提交、加入复习等用户态写入链路，直接复用容易产生 401 串联和边界漂移。

匿名详情页允许：

- 读取公开场景。
- 展示场景文本、句子列表、chunk/detail 信息。
- 播放已生成 TTS。
- 使用现有允许的匿名 explain 能力和配额。
- 展示预生成练习题。

匿名详情页禁止：

- 生成场景。
- 导入场景。
- 生成或重生成练习题。
- 保存表达。
- 加入复习。
- 提交练习。
- 写入 progress、review、practice run、variant run 等用户态数据。

### 4. 预生成练习题只允许本地体验

匿名练习体验只读取已存在的预生成题集。访客可以选择答案、查看本地即时反馈或本地完成状态，但这些状态只存在于前端内存或 `sessionStorage`，不得写入业务表。

当访客点击提交、保存结果、加入复习或生成更多题目时，系统必须阻断操作并展示注册引导。

若某个精选场景没有预生成题集，页面应显示注册后生成或注册后练习的引导，不得在匿名模式下调用生成接口。

### 5. 写入和生成统一收敛为注册引导

所有匿名不可用动作都应进入统一阻断组件或 toast/modal，文案说明“注册后可解锁保存、复习、生成和学习记录”。前端按钮可以保持可见但受限，避免用户误以为功能不存在。

服务端仍必须作为最终边界：即使前端隐藏或阻断失败，写入和生成类 API 对匿名请求也必须返回受控拒绝，不得继续执行业务写入或上游生成。

## Risks / Trade-offs

- [Risk] `/trial` 与主应用场景页能力重复，后续维护可能分叉。缓解：匿名层只承载只读和注册转化，不承载已登录主链路状态。
- [Risk] 精选场景数据不完整导致 TTS 或练习入口空态过多。缓解：allowlist 必须只包含数据完整的公开场景。
- [Risk] 前端本地练习状态让用户误以为已保存。缓解：练习提交、结果保存、加入复习位置必须出现明确注册提示。
- [Risk] 路由白名单过宽导致主应用匿名可访问。缓解：只新增 `/trial/*`，测试必须覆盖主路由仍跳登录。

## Migration Plan

1. 新增 OpenSpec 草案并通过 strict 校验。
2. 实施 `/trial` 场景列表和 `/trial/scene/[slug]` 只读详情。
3. 接入精选场景 allowlist 和预生成练习题只读读取。
4. 在匿名详情页统一接入写入/生成阻断注册引导。
5. 补充路由守卫、只读渲染、练习不可提交、写 API 仍拒绝匿名的测试。
6. 同步 feature-map、domain-rules、dev-log；完成实现 review 后再 archive。

## Open Questions

- 精选场景 slug 初始名单需要从现有已预生成 TTS 和练习题的数据中确认。
- 预生成练习题目前的数据结构是否可直接按公开场景读取，还是需要新增只读查询 helper。
- 首页“免登录试用”入口在实施后是否直接切到 `/trial`，还是保留一个二级入口同时展示单场景分享。
