## MODIFIED Requirements

### Requirement: 认证与高成本接口改动必须先审计边界与失败保护
维护流程 MUST 在实现任何认证入口、登录跳转规则、接口访问边界、高成本外部调用或统一接口治理能力改动前，先审计请求入口、访问控制、来源校验、限流策略、错误收口与回归测试范围。维护者不得只修补单个 route 或页面判断，而忽略对应的中间件、客户端调用、服务端错误路径和排障追踪能力。若改动同时涉及用户态数据访问边界或关键写入一致性，维护者 MUST 额外盘点 `service role` 使用点、主链路写入点、并发风险和回滚面。

#### Scenario: 准备修改后端接口治理基线
- **WHEN** 维护者准备新增或调整 requestId 追踪、限流、来源校验、统一异常处理等接口治理能力
- **THEN** 必须先识别受影响的 middleware、route、handler、service 和测试入口
- **AND** 必须在 OpenSpec change 中明确哪些接口先接入、哪些范围暂不纳入以及剩余风险

#### Scenario: 准备调整高成本或敏感 API route
- **WHEN** 维护者准备修改会触发模型调用、批量重生成、重解析或用户写入的 API route
- **THEN** 必须先盘点 middleware 边界、route 内权限、输入规模限制、来源校验、错误收口和测试覆盖
- **AND** 不得只修补单个 route 的局部判断而忽略完整请求链路

#### Scenario: 准备收紧用户态数据边界或补关键写入一致性
- **WHEN** 维护者准备调整用户态接口的数据访问方式，或为关键写接口增加幂等、条件更新和并发保护
- **THEN** 必须先识别受影响的主链路写入点、数据库边界、回滚方案和测试入口
- **AND** 不得在未盘点 `service role` 白名单和并发风险前直接修改生产写链路

#### Scenario: 完成接口治理改动后准备收尾
- **WHEN** 维护者完成本轮后端接口治理基线改动
- **THEN** 必须运行受影响的最小测试集合并记录验证结果
- **AND** 必须同步更新 CHANGELOG.md 与相关开发记录
