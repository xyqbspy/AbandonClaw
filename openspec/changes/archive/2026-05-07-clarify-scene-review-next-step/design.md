## Context

当前 Scene / Review 的核心能力已经完成闭环：

- Scene detail 可以承载阅读、听熟、表达保存、practice、variants 与学习状态同步。
- Review 可以区分普通表达复习与场景回补，并支持微回忆、熟悉度、变体改写、完整输出和最终反馈。
- Today / learning dashboard 已消费服务端学习摘要和 review 摘要。

但页面体验上还有一个横切问题：能力越完整，页面越容易把“下一步该做什么”藏进状态面板、辅助按钮或说明文本里。本轮不改变业务语义，而是把已有状态翻译成更明确的用户主路径。

## Goals / Non-Goals

**Goals:**

- Scene 主学习视图直接显示当前训练步骤和主行动，不只依赖浮动训练入口。
- Scene 变体学习页突出学习动作，降低删除、管理类动作的视觉优先级。
- Review 每个阶段聚焦一个主问题和一个主 CTA，来源场景等辅助入口不抢主路径。
- 保留现有状态流、写回流和缓存流，只调整展示层级和文案组织。
- 补齐最小回归测试，防止后续把辅助动作重新提升为主动作。

**Non-Goals:**

- 不改 `user_scene_progress`、practice run / attempt / complete、review submit 的数据语义。
- 不新增数据库字段、API 字段或 AI 评分。
- 不改 Review 调度算法、Scene practice 生成策略、variant 生成策略。
- 不做全站视觉系统重构或全局组件抽象。

## Decisions

### 1. Scene 主路径用“页面内任务条 + 浮动进度入口”双层表达

当前浮动入口适合随时查看训练进度，但不适合作为唯一下一步提示。本轮在 Scene base view 的主学习区域增加轻量任务条，展示：

- 当前步骤标题。
- 一句用户可理解的下一步说明。
- 当前步骤主 CTA。
- 可选的已完成 / 可复习提示。

浮动入口继续保留，用于进度总览和小屏快捷操作。

替代方案：只强化浮动入口文案。  
不采用原因：浮动入口在视觉上像工具，不足以承担首次进入页面的主任务说明。

### 2. Scene 变体学习页把管理动作降级

变体学习页的主动作应围绕“学习此变体”和“基于此变体继续练习”。删除变体属于管理动作，应从主按钮组降为次级操作，避免用户把页面理解为生成结果管理器。

替代方案：保留同层级按钮但调整颜色。  
不采用原因：颜色只能降低一部分视觉权重，不能解决动作语义混杂。

### 3. Review 阶段面板只强调当前阶段的单一主问题

Review 递进流本身合理，但每一步如果同时展示阶段说明、参考、来源、调度原因、历史统计和多个动作，会让用户觉得像在填流程表。本轮保留必要信息，但呈现顺序改为：

1. 当前阶段主问题。
2. 当前输入 / 选择区域。
3. 必要提示或参考。
4. 辅助信息。
5. 底部主 CTA。

替代方案：减少阶段数量。  
不采用原因：阶段数量是本轮正式化后的核心训练结构，减少阶段会改变训练语义。

### 4. 来源场景入口保持辅助层级

来源场景入口有价值，但它不是 Review 当前阶段的主动作。本轮保留可访问来源场景、场景回补回到练习等能力，但视觉层级应低于“继续当前复习阶段”。

替代方案：隐藏来源入口。  
不采用原因：来源回看是复习降级和语境恢复的重要辅助能力，不能直接移除。

## Risks / Trade-offs

- [Risk] Scene 新增任务条可能与浮动入口重复。  
  → Mitigation：任务条只展示当前主动作，浮动入口展示完整进度，两者职责分开。

- [Risk] Review 文案收短后，用户可能不理解正式信号边界。  
  → Mitigation：保留“不是 AI 评分 / 只记录训练信号”等关键边界，但移到辅助说明，不占主问题位置。

- [Risk] 辅助入口降级可能降低来源场景回看发现率。  
  → Mitigation：保留入口，只降低视觉优先级；场景回补任务仍可提供回到场景练习入口。

- [Risk] 测试只验证存在性，难以验证“用户感知”。  
  → Mitigation：测试覆盖主 CTA 文案、辅助动作层级、阶段切换和不破坏 submit / route 行为；体验判断记录在 design 与 dev-log。

## Migration Plan

1. 先实现纯展示与文案层级调整，不改后端接口。
2. 跑 Scene / Review 相关 interaction 和 regression 测试。
3. 同步 feature-flow / system-design 文档和 dev-log。
4. 若上线后发现任务条与浮动入口重复，再优先收敛文案，不回退状态流。

## Open Questions

- Scene 主任务条是否只在 base scene view 展示，还是在 practice / variants view 也展示轻量返回提示？
- Review 来源场景入口是保持当前页面外部区域，还是移入阶段面板底部辅助区？
- 变体删除动作降级为“更多”菜单还是危险次按钮，由实现时结合现有组件最小化决定。

## Stability Closure

### 本轮收口

- 把 Scene 下一步从浮动进度入口扩展到主学习视图。
- 把 Review 阶段说明从“完整解释”收敛为“当前主问题 + 必要提示”。
- 把来源场景、删除变体等辅助动作明确降级。

### 本轮不收

- 不把 Review 的改写 / 完整输出引入 AI 质量评分。
- 不调整 Scene / Review 的服务端状态和调度规则。
- 不抽全局页面骨架组件。

### 风险记录

- 若后续继续出现跨页面主 CTA 层级漂移，应进入 `learning-action-button-hierarchy` 或 component-library 专项。
- 若后续要改变 Review 阶段数量或调度语义，应另起 Review 训练语义 change。
